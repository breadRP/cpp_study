# C++ 상속, public, virtual, 오버라이딩, 다형성 정리

## 1. C++에서 상속할 때 public을 붙이는 이유

C++에서 클래스 상속을 할 때 보통 다음처럼 작성한다.

~~~cpp
class Dog : public Animal {
};
~~~

여기서 `public Animal`의 의미는 다음과 같다.

> `Dog`는 외부에서도 `Animal`의 한 종류로 취급될 수 있다.

즉, `Dog` 객체를 `Animal` 타입의 포인터나 참조로 받을 수 있다.

~~~cpp
Dog dog;

Animal* a = &dog;   // 가능
Animal& r = dog;    // 가능
~~~

이것이 가능한 이유는 `Dog`가 `Animal`을 `public` 상속했기 때문이다.

---

## 2. public을 안 붙이면 어떻게 되는가?

C++에서 `class`는 상속 접근 지정자를 생략하면 기본값이 `private`이다.

~~~cpp
class Dog : Animal {
};
~~~

위 코드는 사실상 다음과 같다.

~~~cpp
class Dog : private Animal {
};
~~~

이 경우 외부에서는 `Dog`를 `Animal`로 취급할 수 없다.

~~~cpp
Dog dog;

Animal* a = &dog; // 오류
~~~

즉, `public` 상속은 외부에서 자식 클래스를 부모 클래스처럼 사용할 수 있게 해주고, `private` 상속은 부모 클래스의 기능을 내부 구현용으로만 사용하는 것에 가깝다.

---

## 3. is-a 관계란?

`is-a` 관계는 다음처럼 말했을 때 자연스러운 관계를 의미한다.

> Dog is an Animal.  
> 개는 동물이다.

예를 들어 다음은 자연스럽다.

~~~cpp
class Dog : public Animal {
};

class Cat : public Animal {
};

class Student : public Person {
};
~~~

각각 다음처럼 읽을 수 있다.

- `Dog`는 `Animal`이다.
- `Cat`은 `Animal`이다.
- `Student`는 `Person`이다.

이런 경우에는 `public` 상속을 사용하는 것이 자연스럽다.

반대로 다음은 어색하다.

~~~cpp
class Car : public Engine {
};
~~~

이 코드는 다음처럼 읽힌다.

> 자동차는 엔진이다.

하지만 실제로는 자동차가 엔진인 것이 아니라, 자동차가 엔진을 가지고 있는 것이다.

이런 경우에는 상속보다 포함을 사용하는 것이 자연스럽다.

~~~cpp
class Car {
private:
    Engine engine;
};
~~~

## 4. has-a 관계란?

`has-a` 관계는 어떤 클래스가 다른 객체를 멤버 변수로 가지고 있는 관계이다.

예를 들어:

~~~cpp
class Engine {
public:
    void start() {
        // engine start
    }
};

class Car {
private:
    Engine engine;
};
~~~

이 관계는 다음처럼 읽는다.

> Car has an Engine.  
> 자동차는 엔진을 가지고 있다.

즉, `Car`는 `Engine`의 한 종류가 아니라, 내부에 `Engine` 객체를 가지고 있는 것이다.

정리하면 다음과 같다.

| 관계 | 의미 | 구현 방식 |
|---|---|---|
| is-a | A는 B의 한 종류다 | `class A : public B` |
| has-a | A는 B를 가지고 있다 | `class A { B b; }` |

## 5. 부모 타입으로 자식 객체를 받는 이유

`Dog`라는 구체적인 클래스가 있는데도 굳이 `Animal` 타입으로 받는 이유는 일반성 때문이다.

예를 들어 다음 함수는 `Dog`만 받을 수 있다.

~~~cpp
void makeSound(Dog& dog) {
    dog.sound();
}
~~~

이 함수는 `Dog` 전용 함수이다.

~~~cpp
Dog dog;
makeSound(dog); // 가능

Cat cat;
makeSound(cat); // 불가능
~~~

하지만 다음처럼 부모 타입인 `Animal&`로 받으면:

~~~cpp
void makeSound(Animal& animal) {
    animal.sound();
}
~~~

`Animal`을 상속한 여러 자식 클래스를 받을 수 있다.

~~~cpp
Dog dog;
Cat cat;

makeSound(dog); // 가능
makeSound(cat); // 가능
~~~

즉, 부모 타입으로 받는 이유는 다음과 같다.

> 구체적인 `Dog`, `Cat`에 각각 의존하지 않고,  
> `Animal`이라는 공통 타입으로 여러 객체를 한 번에 다루기 위해서이다.

---

## 6. 부모 타입으로 받으면 어떤 기능을 쓸 수 있는가?

다음 코드를 보자.

~~~cpp
Dog dog;
Animal* a = &dog;
~~~

여기서 실제 객체는 `Dog`이다.

하지만 변수 `a`의 타입은 `Animal*`이다.

따라서 `a`를 통해서는 기본적으로 `Animal`에 정의된 기능만 사용할 수 있다.

~~~cpp
class Animal {
public:
    void eat() {
        cout << "Animal eats" << endl;
    }
};

class Dog : public Animal {
public:
    void bark() {
        cout << "Dog barks" << endl;
    }
};

int main() {
    Dog dog;

    Animal* a = &dog;

    a->eat();   // 가능
    a->bark();  // 오류
}
~~~

`bark()`는 `Dog`에만 있고 `Animal`에는 없기 때문에, `Animal*` 타입의 변수로는 호출할 수 없다.

즉:

| 표현 | 의미 |
|---|---|
| 실제 객체 | `Dog` |
| 변수 타입 | `Animal*` |
| 호출 가능한 함수 | `Animal`에 선언된 함수 |
| `Dog` 전용 함수 | 직접 호출 불가 |

단, `virtual` 함수는 예외적으로 실제 객체 기준으로 실행될 수 있다.

## 7. 오버라이딩이란?

오버라이딩은 부모 클래스에 있는 함수를 자식 클래스에서 다시 정의하는 것이다.

~~~cpp
class Animal {
public:
    void sound() {
        cout << "Animal sound" << endl;
    }
};

class Dog : public Animal {
public:
    void sound() {
        cout << "Bark" << endl;
    }
};
~~~

여기서 `Dog`는 `Animal`의 `sound()` 함수를 다시 정의했다.

이것을 오버라이딩이라고 한다.

오버라이딩의 목적은 다음과 같다.

> 부모가 제공하는 공통 함수 이름은 유지하되,  
> 자식 클래스마다 다른 동작을 하도록 만드는 것

예를 들어:

~~~cpp
class Dog : public Animal {
public:
    void sound() {
        cout << "Bark" << endl;
    }
};

class Cat : public Animal {
public:
    void sound() {
        cout << "Meow" << endl;
    }
};
~~~

`Dog`와 `Cat`은 둘 다 `sound()`라는 같은 이름의 함수를 가지지만, 실제 동작은 다르다.

## 8. virtual이란?

`virtual`은 부모 클래스의 함수 앞에 붙이는 키워드이다.

의미는 다음과 같다.

> 부모 타입으로 함수를 호출하더라도,  
> 실제 객체가 자식 객체라면 자식 클래스의 오버라이딩된 함수를 실행하라.

예를 들어 `virtual`이 없는 경우를 보자.

~~~cpp
class Animal {
public:
    void sound() {
        cout << "Animal sound" << endl;
    }
};

class Dog : public Animal {
public:
    void sound() {
        cout << "Bark" << endl;
    }
};

void makeSound(Animal& animal) {
    animal.sound();
}

int main() {
    Dog dog;
    makeSound(dog);
}
~~~

이 경우 출력은 다음과 같다.

~~~text
Animal sound
~~~

왜냐하면 `makeSound()` 안에서 매개변수 타입이 `Animal&`이기 때문이다.

컴파일러는 `animal.sound()`를 보고 `Animal`의 `sound()`를 호출한다.

---

이번에는 부모 함수에 `virtual`을 붙여보자.

~~~cpp
class Animal {
public:
    virtual void sound() {
        cout << "Animal sound" << endl;
    }
};

class Dog : public Animal {
public:
    void sound() override {
        cout << "Bark" << endl;
    }
};

void makeSound(Animal& animal) {
    animal.sound();
}

int main() {
    Dog dog;
    makeSound(dog);
}
~~~

이 경우 출력은 다음과 같다.

~~~text
Bark
~~~

`makeSound()`의 매개변수 타입은 여전히 `Animal&`이다.

하지만 실제로 들어온 객체는 `Dog`이다.

부모 클래스의 `sound()`가 `virtual`이기 때문에, 실제 객체인 `Dog`의 `sound()`가 실행된다.

---

## 9. override란?

`override`는 자식 클래스에서 사용하는 키워드이다.

~~~cpp
class Dog : public Animal {
public:
    void sound() override {
        cout << "Bark" << endl;
    }
};
~~~

`override`의 의미는 다음과 같다.

> 이 함수는 부모 클래스의 virtual 함수를 오버라이딩한 것이다.

`override`는 필수는 아니지만 쓰는 것이 좋다.

왜냐하면 함수 이름이나 매개변수를 실수했을 때 컴파일러가 오류를 잡아주기 때문이다.

예를 들어:

~~~cpp
class Animal {
public:
    virtual void sound() {
    }
};

class Dog : public Animal {
public:
    void sound(int x) override {
    }
};
~~~

이 코드는 오류가 난다.

부모 클래스의 함수는 다음과 같다.

~~~cpp
void sound()
~~~

그런데 자식 클래스의 함수는 다음과 같다.

~~~cpp
void sound(int x)
~~~

매개변수 형태가 다르기 때문에 오버라이딩이 아니다.

`override`를 붙이면 이런 실수를 컴파일러가 잡아준다.

## 10. 다형성이란?

다형성은 같은 코드가 실제 객체의 종류에 따라 다르게 동작하는 성질이다.

예를 들어:

~~~cpp
class Animal {
public:
    virtual void sound() {
        cout << "Animal sound" << endl;
    }
};

class Dog : public Animal {
public:
    void sound() override {
        cout << "Bark" << endl;
    }
};

class Cat : public Animal {
public:
    void sound() override {
        cout << "Meow" << endl;
    }
};

void makeSound(Animal& animal) {
    animal.sound();
}

int main() {
    Dog dog;
    Cat cat;

    makeSound(dog);
    makeSound(cat);
}
~~~

출력은 다음과 같다.

~~~text
Bark
Meow
~~~

`makeSound()` 함수는 하나뿐이다.

~~~cpp
void makeSound(Animal& animal) {
    animal.sound();
}
~~~

하지만 실제로 전달된 객체가 `Dog`이면 `Dog::sound()`가 실행되고, `Cat`이면 `Cat::sound()`가 실행된다.

이것이 다형성이다.

---

## 11. 상속, 오버라이딩, virtual, 다형성의 관계

이 네 개념은 보통 함께 사용된다.

### 상속

~~~cpp
class Dog : public Animal {
};
~~~

`Dog`를 `Animal`의 한 종류로 다룰 수 있게 한다.

즉:

~~~cpp
Animal& animal = dog;
Animal* ptr = &dog;
~~~

이 가능해진다.

### 오버라이딩

~~~cpp
void sound() override {
    cout << "Bark" << endl;
}
~~~

부모 클래스의 함수를 자식 클래스에서 자기 방식으로 다시 정의한다.

### virtual

~~~cpp
virtual void sound()
~~~

부모 타입으로 호출해도 실제 자식 객체의 함수가 실행되게 한다.

### 다형성

~~~cpp
void makeSound(Animal& animal) {
    animal.sound();
}
~~~

같은 코드가 실제 객체의 종류에 따라 다르게 동작한다.

## 12. 부모 타입 함수는 어디에 구현하는가?

다음 함수는 클래스 안에 넣는 멤버 함수가 아니라, 보통 클래스 밖에 따로 만드는 일반 함수이다.

~~~cpp
void makeSound(Animal& animal) {
    animal.sound();
}
~~~

전체 구조는 다음과 같다.

~~~cpp
#include <iostream>
using namespace std;

class Animal {
public:
    virtual void sound() {
        cout << "Animal sound" << endl;
    }
};

class Dog : public Animal {
public:
    void sound() override {
        cout << "Bark" << endl;
    }
};

class Cat : public Animal {
public:
    void sound() override {
        cout << "Meow" << endl;
    }
};

// 클래스 밖에 있는 일반 함수
void makeSound(Animal& animal) {
    animal.sound();
}

int main() {
    Dog dog;
    Cat cat;

    makeSound(dog);
    makeSound(cat);

    return 0;
}
~~~

출력:

~~~text
Bark
Meow
~~~

즉, `makeSound()`는 `Animal`, `Dog`, `Cat`의 멤버 함수가 아니다.

그냥 `Animal&`을 인자로 받는 독립적인 함수이다.

## 13. 자식 클래스를 인자로 주면 어떤 함수가 호출되는가?

다음과 같은 함수가 있다고 하자.

~~~cpp
void makeSound(Animal& animal) {
    animal.sound();
}
~~~

여기에 `Dog` 객체를 넘긴다.

~~~cpp
Dog dog;
makeSound(dog);
~~~

이때 어떤 함수가 호출되는지는 부모 클래스의 `sound()`가 `virtual`인지에 따라 달라진다.

| 조건 | 호출되는 함수 |
|---|---|
| `Animal::sound()`가 `virtual`이다 | `Dog::sound()` |
| `Animal::sound()`가 `virtual`이 아니다 | `Animal::sound()` |
| `Dog`가 `sound()`를 오버라이딩하지 않았다 | `Animal::sound()` |

즉, 자식 클래스의 오버라이딩된 함수를 호출하려면 부모 클래스 함수에 `virtual`이 필요하다.

## 14. 추상 클래스와 순수 가상 함수

부모 클래스가 직접 동작을 구현하지 않고, 자식 클래스가 반드시 구현하도록 강제할 수도 있다.

이때 순수 가상 함수를 사용한다.

~~~cpp
class Animal {
public:
    virtual void sound() = 0;
};
~~~

여기서 `= 0`은 다음 의미이다.

> 이 함수는 부모 클래스에서 구현하지 않는다.  
> 자식 클래스가 반드시 오버라이딩해야 한다.

이런 함수를 가진 클래스를 추상 클래스라고 한다.

~~~cpp
class Animal {
public:
    virtual void sound() = 0;
};

class Dog : public Animal {
public:
    void sound() override {
        cout << "Bark" << endl;
    }
};
~~~

추상 클래스는 직접 객체를 만들 수 없다.

~~~cpp
Animal animal; // 오류
~~~

하지만 포인터나 참조 타입으로는 사용할 수 있다.

~~~cpp
Dog dog;

Animal& animal = dog;
animal.sound(); // Bark
~~~


의미:

> Dog는 Animal의 한 종류이다.

그래서 다음이 가능하다.

~~~cpp
Animal* a = &dog;
Animal& r = dog;
~~~

---

### 부모 타입으로 받는 이유

~~~cpp
void makeSound(Animal& animal)
~~~

의미:

> Dog, Cat 같은 구체적인 클래스에 각각 의존하지 않고,  
> Animal이라는 공통 타입으로 묶어서 처리하겠다.

---

### 오버라이딩

~~~cpp
void sound() override
~~~

의미:

> 부모의 함수를 자식 클래스에서 다시 정의한다.

---

### virtual

~~~cpp
virtual void sound()
~~~

의미:

> 부모 타입으로 호출하더라도 실제 객체가 자식이면 자식의 함수가 실행되게 한다.

---

### 다형성


같은 함수 `makeSound()`를 호출했지만 실제 객체가 다르기 때문에 다른 결과가 나온다.

이것이 다형성이다.

## 16. 한 문장으로 최종 정리

C++에서 `public` 상속은 자식 객체를 부모 타입으로 다룰 수 있게 해주고, 오버라이딩은 자식 클래스마다 같은 함수를 다르게 동작하게 만들며, `virtual`은 부모 타입으로 호출해도 실제 자식 객체의 함수가 실행되게 해준다. 이 조합을 통해 같은 코드가 여러 객체에 대해 다르게 동작하는 다형성을 구현할 수 있다.
